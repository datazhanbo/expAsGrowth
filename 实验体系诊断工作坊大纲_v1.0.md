# 实验体系诊断工作坊（2天）

> 展博增长实验室 · 交付标准
>
> 目标：用2天时间，完成团队实验能力的全面诊断，对齐问题根因，输出可落地的30/60/90天行动路线图

---

## 工作坊前置准备

### 客户侧准备
1. **参与人员（建议6-10人）**
   - 必须到场：产品负责人、增长负责人、数据负责人、技术负责人
   - 可选到场：业务线负责人、制作人、CTO/VP等管理层（第二天下午到场听结论）

2. **数据准备**
   - 过去3个月的实验清单（如果有）
   - 当前的实验平台截图/演示（如果有）
   - 核心指标的口径定义文档
   - 组织架构图

3. **场地要求**
   - 有白板/投影
   - 可以分组讨论
   - 午餐和休息区安排

### 顾问侧准备
1. 提前阅读客户提供的所有材料
2. 准备诊断工具包Excel模板
3. 准备七个常见败局案例
4. 准备白板贴纸、便签等物料

---

## 第一天：现状诊断与问题对齐

### 上午 9:00-12:00 六维度能力评估
**目标**：对实验体系的六大模块做全面打分，建立对现状的统一认知

| 时间 | 环节 | 内容 | 产出 |
|------|------|------|------|
| 9:00-9:30 | 开场破冰 | 1. 工作坊目标与议程介绍<br>2. "为什么你建的实验平台没人信？" - 开场痛点分享<br>3. 每个人用1分钟说："我觉得我们实验最大的问题是____" | 痛点收集白板 |
| 9:30-10:30 | 分组评分：技术模块 | 分成2组，分别评分：<br>• 组A：流量管理 + 工程架构<br>• 组B：统计分析 + 指标体系<br><br>每组先独立评分（20分钟），然后组内讨论对齐（30分钟） | 4个模块初步评分 |
| 10:30-10:45 | 茶歇 | | |
| 10:45-11:45 | 分组评分：管理模块 + 交叉评审 | 继续评分：<br>• 全体：实验管理 + 组织文化<br><br>然后交叉评审：<br>• 组A给组B的两个模块提疑问<br>• 组B给组A的两个模块提疑问<br>• 有分歧的地方充分讨论 | 6个模块评分对齐 |
| 11:45-12:00 | 上午小结 | 汇总6个模块得分，展示雷达图<br>标记3个得分最低的模块作为下午重点讨论 | 上午评分汇总 |

### 下午 14:00-17:30 四层阻力根因分析
**目标**：不仅仅知道"分数低"，更要知道"为什么分数低"，找到根本原因

| 时间 | 环节 | 内容 | 产出 |
|------|------|------|------|
| 14:00-14:30 | 四层阻力模型讲解 | 讲解工具层→流程层→组织层→文化层的阻力模型<br>讲解每个层级的典型症状和突破方法 | 四层阻力白板图 |
| 14:30-15:00 | "七个败局"对标 | 对照七个常见败局，每个问题举团队自己的例子<br>• 互斥桶陷阱？<br>• p值崇拜？<br>• 偷看数据？<br>• 指标迷宫？<br>• 没有护栏？<br>• 没有沉淀？<br>• 组织孤岛？ | 败局对标结果 |
| 15:00-15:15 | 茶歇 | | |
| 15:15-16:30 | 根因分析工作坊（5 Whys） | 针对上午得分最低的3个模块，每个模块做5 Whys分析：<br><br>例："我们的指标体系分数低"<br>• 为什么？因为不同部门口径不一样<br>• 为什么口径不一样？因为没有统一的指标字典<br>• 为什么没有？因为没人牵头做<br>• 为什么没人牵头？因为这不是任何人的KPI<br>• 为什么不是？因为管理层没觉得这是优先级<br><br>每个模块用一张白板，把所有原因写下来 | 3张根因分析白板 |
| 16:30-17:00 | 阻力层归类 | 把所有根因归类到四层阻力：<br>• 哪些是工具层问题？（可以用技术解决的）<br>• 哪些是流程层问题？（可以用制度解决的）<br>• 哪些是组织层问题？（需要调整权责的）<br>• 哪些是文化层问题？（需要管理层改变的）<br><br>统计每一层的问题数量，找到主要卡点 | 四层阻力分布图 |
| 17:00-17:30 | 第一天小结 | 确认："我们团队主要卡在第X层阻力"<br>列出TOP 5最痛的问题<br>布置第二天作业：每个人晚上想"如果只能解决一个问题，你先解决什么" | 第一天结论页 |

## 第二天：解决方案与路线图

### 上午 9:00-12:00 解决方案设计
**目标**：针对核心问题，讨论具体的解决方案，而不是空泛的"我们要改进"

| 时间 | 环节 | 内容 | 产出 |
|------|------|------|------|
| 9:00-9:30 | 第二天开场 | 回顾第一天的评分和根因分析<br>确认大家对"我们卡在哪一层"有没有共识 | 共识确认 |
| 9:30-10:30 | 解决方案头脑风暴 | 针对TOP 5最痛的问题，每个问题用20分钟讨论：<br><br>1. 理想状态是什么样的？<br>2. 我们现在离理想状态差什么？<br>3. 具体做什么能补上这个差距？<br>4. 谁来做？<br><br>用"怎么做"代替"做什么" | 5个问题的解决方案 |
| 10:30-10:45 | 茶歇 | | |
| 10:45-11:45 | 优先级排序 | 把所有解决方案列出来，用两个维度打分：<br>• 影响程度：解决这个问题，效果有多大？（1-5分）<br>• 实施难度：做这个事情，难不难？（1-5分）<br><br>然后画四象限：<br>• 第一象限：高影响、低难度 → 马上做（30天）<br>• 第二象限：高影响、高难度 → 规划做（60-90天）<br>• 第三象限：低影响、低难度 → 有空做<br>• 第四象限：低影响、高难度 → 不做 | 四象限优先级图 |
| 11:45-12:00 | 上午小结 | 确认第一象限（马上做）的3-5个行动项<br>确认第二象限（规划做）的3-5个行动项 | 优先级确认 |

### 下午 14:00-17:30 路线图输出 + 管理层汇报
**目标**：输出可执行的30/60/90天路线图，并向管理层汇报，获得资源承诺

| 时间 | 环节 | 内容 | 产出 |
|------|------|------|------|
| 14:00-15:00 | 30天行动项细化 | 针对第一象限的行动项，每个行动项写清楚：<br>• 具体做什么（要可验证，不是"改进数据质量"，而是"核心10个指标的数据准确率提升到95%以上"）<br>• 负责人是谁<br>• 需要什么资源<br>• 怎么验证做完了<br>• 可能遇到什么风险 | 30天行动计划表 |
| 15:00-15:15 | 茶歇 | | |
| 15:15-16:00 | 60/90天路线图 | 针对第二象限的行动项，排到60天和90天：<br><br>• 60天目标：流程跑通，能力建立<br>• 90天目标：规模化，文化开始形成<br><br>同时输出团队的成熟度提升目标：<br>• 现在：L? 总分：???<br>• 90天后：目标L? 总分：???（提升50分左右是合理目标） | 60/90天路线图 |
| 16:00-16:30 | 汇报材料准备 | 整理两天的产出，准备给管理层的汇报：<br>1. 我们现在在哪里？（评分雷达图）<br>2. 我们的核心问题是什么？（四层阻力和七个败局）<br>3. 我们准备怎么做？（四象限优先级）<br>4. 我们需要什么支持？（资源和授权） | 汇报PPT大纲 |
| 16:30-17:30 | 管理层汇报 + 确认 | 向管理层（CTO/VP/制作人等）汇报两天的结论<br>重点讨论：<br>• 管理层是否认可问题诊断？<br>• 行动项的优先级是否需要调整？<br>• 需要管理层提供什么资源和授权？<br>• 90天之后我们怎么评估成功？ | 管理层签字确认的路线图 |

## 工作坊交付物清单

### 工作坊结束后3个工作日内交付
1. ✅ **《实验能力六维度诊断报告》**
   - 六个模块的详细评分和雷达图
   - 每个模块的优势和不足分析
   - 与行业基准的对比

2. ✅ **《根因分析与四层阻力定位报告》**
   - 七个败局对标结果
   - 核心问题的5 Whys分析
   - 团队当前主要卡在第几层阻力的结论

3. ✅ **《30/60/90天行动路线图》**
   - 30天：高影响低难度行动项清单（含负责人、验证标准）
   - 60-90天：系统建设规划
   - 成熟度提升目标

4. ✅ **《工作坊现场照片与纪要》**
   - 所有白板讨论的照片
   - 关键讨论点的文字记录
   - 分歧点和最终结论

## 工作坊成功标准

工作坊结束时，如果以下三条都满足，就是成功的：

1. **共识达成**：所有参与人对"我们的实验体系主要卡在哪一层、最痛的问题是什么"没有本质分歧

2. **行动明确**：30天的行动项是具体的、可验证的、有负责人的，不是"我们要改进"这种空话

3. **管理层支持**：管理层认可诊断结论，同意路线图，并且承诺了必要的资源和授权

## 常见风险与应对

| 风险 | 应对方式 |
|------|----------|
| 讨论跑偏，变成吐槽大会 | 主持人及时拉回，"这个问题我们记下来，现在先聚焦在有解的问题上" |
| 分数虚高，大家都不想承认问题 | 顾问带头"挑刺"，用其他客户的案例做对标，让大家看到差距 |
| 管理层不来听汇报 | 提前和对接人强调：管理层不到场，工作坊效果打5折 |
| 团队太大，意见太分散 | 控制人数在6-10人，超过10人就分场次 |
| 技术问题和组织问题混在一起 | 用四层阻力模型强行分类，先分清楚再讨论 |
